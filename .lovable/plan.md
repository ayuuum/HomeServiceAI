
# タブレットモードでのメニュー崩れ修正

## 問題の分析

### 現状
- **モバイル（768px未満）**: ハンバーガーメニュー（MobileNav）が表示される ✓
- **デスクトップ（768px以上）**: 8つのナビリンクが水平に並ぶ

### 問題点
タブレットサイズ（768px〜1024px）では、水平ナビゲーションが表示されますが、8つのリンクを横に並べるには幅が足りず、テキストやボタンが潰れてしまいます。

```text
【現状】タブレット（768px〜1024px）
┌────────────────────────────────────────────────────────────────────┐
│ [☰] ロゴ  ダッシュ顧客管予約管受信ト一斉配経営管予約ペ設定 [出] │
│                    ↑ テキストが潰れる                              │
└────────────────────────────────────────────────────────────────────┘
```

## 解決策

ブレークポイントを `md`（768px）から `lg`（1024px）に変更し、タブレットサイズでもハンバーガーメニューを使用するようにします。

```text
【改善後】タブレット（768px〜1024px）
┌────────────────────────────────────────┐
│ [☰] ロゴ                          [出] │  ← モバイルと同様のハンバーガーメニュー
└────────────────────────────────────────┘

【改善後】デスクトップ（1024px以上）
┌──────────────────────────────────────────────────────────────────────────────┐
│ [☰] ロゴ  ダッシュボード 顧客管理 予約管理 受信トレイ 一斉配信 経営管理... [出] │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 変更ファイル

| ファイル | 変更内容 |
|----------|----------|
| `src/components/AdminHeader.tsx` | ナビゲーションの表示条件を `md:flex` → `lg:flex` に変更 |
| `src/components/MobileNav.tsx` | ハンバーガーボタンの非表示条件を `md:hidden` → `lg:hidden` に変更 |

## 変更詳細

### AdminHeader.tsx
```tsx
// Line 50: md:flex → lg:flex
<nav className="hidden lg:flex items-center gap-1 ml-6">

// Line 124: hidden sm:flex → hidden lg:flex（ログアウトボタン）
<Button variant="outline" size="icon" className="hidden lg:flex">
```

### MobileNav.tsx
```tsx
// Line 34: md:hidden → lg:hidden
<Button variant="ghost" size="icon" className="lg:hidden">
```

## ブレークポイント対応表（変更後）

| 画面幅 | デバイス | ナビゲーション表示 |
|--------|----------|-------------------|
| 〜767px | スマートフォン | ハンバーガーメニュー |
| 768px〜1023px | タブレット | ハンバーガーメニュー |
| 1024px〜 | デスクトップ | 水平ナビゲーション |

## メリット
1. **タブレットでの視認性向上**: 潰れずに読めるナビゲーション
2. **タップターゲット確保**: 十分なサイズのボタン/リンク
3. **一貫したUX**: タブレットユーザーも使いやすいハンバーガーメニュー

## 実装の安全性
- CSSクラス名の変更のみで、ロジックへの影響なし
- Tailwindの標準ブレークポイント（lg: 1024px）を使用
