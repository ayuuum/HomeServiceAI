

# 設定メニューの簡素化プラン

## 現状の問題

現在の設定ページには**6つのタブ**が横並びで、特にモバイルでは：
- タブが潰れて読みにくい
- カテゴリの境界が曖昧（「予約ページ」と「サービス」は関連が深い）
- 全体像が把握しづらい

```text
現在: [予約] [時間] [サービス] [設定] [LINE] [決済]  ← 6つは多すぎ
```

## 改善案: 3タブ構成に統合

関連する設定をグループ化し、タブを3つに減らします：

```text
改善後: [店舗設定] [連携・決済] [アカウント]
```

| 新タブ | 統合される旧タブ | 含む内容 |
|--------|-----------------|---------|
| 店舗設定 | 予約ページ + 営業時間 + サービス | QR、ブランディング、営業時間、サービス管理 |
| 連携・決済 | LINE連携 + 決済 | LINE設定ウィザード、Stripe連携、決済ON/OFF |
| アカウント | アカウント | プロフィール、パスワード、メール変更、組織設定 |

### 店舗設定タブの中身（セクション分け）

各セクションはCardで区切り、折りたたみ可能にします：

```text
店舗設定
├── 予約ページ（QRコード・URL）
├── ブランディング（ロゴ・カラー・見出し）
├── 営業時間
└── サービス管理
```

### 連携・決済タブの中身

```text
連携・決済
├── LINE連携（ウィザード or 設定フォーム）
└── オンライン決済（Stripe連携・決済ON/OFF）
```

## 技術的な変更

### 変更ファイル

**`src/pages/ProfilePage.tsx`**のみ変更：

1. TabsListを `grid-cols-6` から `grid-cols-3` に変更
2. TabsContentを3つに統合：
   - `store`: 旧booking + hours + services の内容をまとめる
   - `integrations`: 旧line + payment の内容をまとめる
   - `account`: 旧accountの内容（変更なし）
3. 各セクション間にSeparatorを追加して視覚的に区切る

### タブのラベルとアイコン

```text
店舗設定    → Settings アイコン
連携・決済  → CreditCard アイコン（外部連携の意味を含む）
アカウント  → User アイコン
```

### モバイル表示

3タブなら `grid-cols-3` で十分な幅が確保でき、テキストが潰れません。
