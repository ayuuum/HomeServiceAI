
# ランディングページ作成計画

## 問題の原因

現在 `haukuripro.com/`（ルートURL）にアクセスすると、`BookingPage` が `default` 組織を探しに行きますが、データベースに存在しないため「ページが見つかりません」と表示されています。

## 解決方針

ルートURL（`/`）に新しいランディングページを表示し、予約ページは `/booking/:orgSlug` でのみアクセスできるようにします。

## 実装内容

### 1. ランディングページの作成

**新規ファイル:** `src/pages/LandingPage.tsx`

```text
ページ構成:
┌─────────────────────────────────────┐
│ ヘッダー                            │
│ - ロゴ                              │
│ - ログインリンク                    │
├─────────────────────────────────────┤
│ ヒーローセクション                   │
│ - キャッチコピー                    │
│ - 「予約する」CTAボタン             │
├─────────────────────────────────────┤
│ サービス紹介                        │
│ - エアコン、キッチン、浴室など      │
├─────────────────────────────────────┤
│ 特徴・メリット                      │
│ - プロの技術、安心価格、簡単予約    │
├─────────────────────────────────────┤
│ フッター                            │
│ - プライバシーポリシーへのリンク    │
│ - 利用規約へのリンク               │
│ - コピーライト                     │
└─────────────────────────────────────┘
```

### 2. フッターコンポーネントの作成

**新規ファイル:** `src/components/Footer.tsx`

再利用可能なフッターコンポーネントを作成し、以下を含めます：
- プライバシーポリシーへのリンク（`/privacy`）
- 利用規約へのリンク（`/terms`）
- コピーライト表示

### 3. ルーティングの更新

**変更ファイル:** `src/App.tsx`

```text
変更前:
  /           → BookingPage（エラーになる）
  /booking/:orgSlug → BookingPage

変更後:
  /           → LandingPage（新しいLP）
  /booking    → BookingPage（default組織用）
  /booking/:orgSlug → BookingPage（特定組織用）
```

### 4. ポリシーページの「戻る」リンク修正

**変更ファイル:**
- `src/pages/PrivacyPolicyPage.tsx`
- `src/pages/TermsOfServicePage.tsx`

「戻る」ボタンが新しいランディングページに正しく戻るよう確認します（既に `/` へのリンクなので変更不要）。

## ランディングページのデザイン

- Airbnb/Uberライクなクリーン&信頼感のあるデザイン
- モバイルファースト対応
- 既存のブランドカラー（ダークネイビー）を使用
- 適切な余白とタイポグラフィ
- CTAボタンは目立つ配置

## 変更ファイル一覧

| ファイル | 変更内容 |
|---------|---------|
| `src/pages/LandingPage.tsx` | 新規作成 - ランディングページ |
| `src/components/Footer.tsx` | 新規作成 - 共通フッター |
| `src/App.tsx` | ルーティング更新 |

## 補足事項

- Google OAuth同意画面の設定には、以下のURLを使用できます：
  - プライバシーポリシー: `https://haukuripro.com/privacy`
  - 利用規約: `https://haukuripro.com/terms`
- 予約ページへのアクセスは `/booking/default` または `/booking/:orgSlug` で引き続き可能
