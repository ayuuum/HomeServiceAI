
# 新規予約ボタンのモバイル最適化

## 問題点
現在の「新規予約」ボタンはデフォルトサイズで表示されており、モバイル画面ではページタイトルや他のUI要素に対してバランスが悪く見えています。

## 改善内容

### ボタンサイズのレスポンシブ対応
- モバイル: `size="sm"` でコンパクトに
- デスクトップ: デフォルトサイズを維持

### アイコンとテキストの調整
- モバイルではアイコンのみ、またはよりコンパクトな表示
- デスクトップでは「新規予約」テキスト付き

## 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `src/pages/CalendarPage.tsx` | ボタンのレスポンシブスタイル調整 |

## 変更詳細

### 現在のコード
```tsx
<Button 
    onClick={() => {
        setInitialBookingDate(undefined);
        setNewBookingModalOpen(true);
    }}
    className="shrink-0"
>
    <Icon name="add" size={18} className="mr-1" />
    新規予約
</Button>
```

### 変更後のコード
```tsx
<Button 
    onClick={() => {
        setInitialBookingDate(undefined);
        setNewBookingModalOpen(true);
    }}
    size="sm"
    className="shrink-0 h-8 md:h-9 text-xs md:text-sm px-2.5 md:px-3"
>
    <Icon name="add" size={16} className="md:mr-1" />
    <span className="hidden md:inline">新規予約</span>
    <span className="md:hidden">予約</span>
</Button>
```

## 主な改善点
1. **ボタン高さ**: モバイルでh-8、デスクトップでh-9（他のナビボタンと統一）
2. **テキストサイズ**: モバイルでtext-xs、デスクトップでtext-sm
3. **テキスト内容**: モバイルでは「予約」のみ表示してコンパクトに
4. **アイコンサイズ**: 16pxに縮小
5. **パディング**: モバイルでpx-2.5、デスクトップでpx-3

## 実装の安全性
- 既存のロジックには影響なし
- 他のUIコンポーネントとのバランスを考慮したサイズ設定
